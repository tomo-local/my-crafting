# Step 6：コネクションプーリング（前提知識）

Step 1〜5 ではリクエストのたびにアップストリームへの新しいTCP接続を確立していました。Step 6 では使い終わった接続を捨てずにプールに戻し、次のリクエストで再利用することでレイテンシを下げます。

---

## 1. TCPコネクション確立のコスト

新規TCP接続を確立するたびに3ウェイハンドシェイクが走ります。

```
クライアント                 アップストリーム
    │──── SYN ──────────────────>│   1往復目
    │<─── SYN-ACK ───────────────│
    │──── ACK ──────────────────>│   ここでやっとデータ送信可能
    │──── HTTPリクエスト ────────>│
```

ローカルで数百マイクロ秒、WAN越しだとRTT × 1.5 のオーバーヘッドです。

---

## 2. コネクションプールの基本設計

プールは「idle（使用されていない）接続のキュー」です。

```
プールの状態:
    idle: [conn1, conn2, conn3]  ← 使い終わって返却された接続

リクエスト到着時:
    idle が空でない → キューから取り出して使う（ハンドシェイク不要）
    idle が空       → 新規接続を確立する

リクエスト完了時:
    レスポンスヘッダーが Connection: close → 接続を捨てる
    そうでない       → プールに返却する
```

---

## 3. プールの上限管理

idle接続を無制限に溜めるとメモリが枯渇します。また、アップストリームにも同時接続数の上限があります。

```
maxIdle:  プールに保持するidle接続の最大数（例: 10）
          → 超えた接続は返却時に捨てる

maxTotal: 全体（idle + active）の最大接続数（Step 6 では省略可）
```

---

## 4. idle接続の有効期限

アップストリームサーバーが一定時間使われていない接続を閉じることがあります（サーバー側のKeepaliveタイムアウト）。プールから取り出した接続が既に切れている（半開き接続）場合があります。

```
検知方法: 書き込み時に error が返ってきたら接続が切れている
対処方法: その接続を捨てて、新規接続を確立して再試行する
```

Nginxのデフォルトの `keepalive_timeout` は 75秒です。プールの接続をそれより長く保持しないよう、TTLを設定するのが安全です。

---

## 5. チャネルを使ったプール実装

Goでは `sync.Pool` を使う方法もありますが、ネットワーク接続のプールには適していません（`sync.Pool` はGCのたびに中身が破棄される可能性がある）。

代わりにバッファ付きチャネルをキューとして使う方法がシンプルです。

```go
type ConnPool struct {
    idle chan net.Conn
    addr string
}

// idle チャネルのキャパシティ = maxIdle
pool := &ConnPool{
    idle: make(chan net.Conn, maxIdle),
    addr: "localhost:9001",
}
```

- **取得**: `select` でチャネルから取り出す。空なら `net.Dial` で新規作成
- **返却**: チャネルが満杯なら `conn.Close()` で捨てる

---

## 📌 まとめ：Step 6 のフロー

**接続取得時:**
1. `select { case conn := <-pool.idle: return conn, nil; default: }` でidle接続を取り出す
2. idle が空なら `net.Dial` で新規接続する

**転送完了時:**
1. レスポンスの `Connection: close` ヘッダーを確認する
2. `close` ならそのまま `conn.Close()` する
3. `close` でなければ `select { case pool.idle <- conn: default: conn.Close() }` で返却する（満杯なら捨てる）

**接続が切れていた場合:**
1. `req.Write(upstreamConn)` がエラーを返したら、その接続を捨てる
2. 新規接続を確立して1回だけリトライする
