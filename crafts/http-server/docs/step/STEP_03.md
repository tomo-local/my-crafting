# Step 3：リクエストラインのパースとルーティング（前提知識）

Step 2 では何が来ても `Hello, World!` を返していました。Step 3 では届いた文字列を分解し、**パスに応じてレスポンスを変える**ルーティングを実装します。

---

## 1. リクエストラインの構造

HTTP リクエストの 1 行目（リクエストライン）は次の形式です。

```
GET /about HTTP/1.1\r\n
```

スペースで区切ると 3 つのフィールドになります。

| フィールド | 例 | 意味 |
|---|---|---|
| メソッド | `GET` | 操作の種類 |
| パス | `/about` | リソースの場所 |
| バージョン | `HTTP/1.1` | プロトコルのバージョン |

---

## 2. パース手順

リクエスト全体の先頭から 1 行目だけを取り出し、スペースで分割します。

```
受信バイト列
    ↓ string() で変換
"GET /about HTTP/1.1\r\nHost: localhost:8080\r\n..."
    ↓ strings.Split(text, "\r\n") で行分割
["GET /about HTTP/1.1", "Host: localhost:8080", ...]
    ↓ [0] 番目がリクエストライン
"GET /about HTTP/1.1"
    ↓ strings.Fields() でスペース分割
["GET", "/about", "HTTP/1.1"]
```

`strings.Fields` は連続する空白をまとめて分割するため、ブラウザが付加する余分なスペースに対しても安全です。

---

## 3. ルーティングのロジック

パスを switch または if-else で分岐します。Step 3 での最低限の要件は以下の 3 パターンです。

| パス | ステータス | ボディ |
|---|---|---|
| `/` | 200 OK | トップページ |
| `/about` | 200 OK | About ページ |
| それ以外 | 404 Not Found | Not Found |

---

## 4. 404 レスポンスの書式

ステータスコードが違うだけで形式は同じです。

```
HTTP/1.1 404 Not Found\r\n
Content-Length: 9\r\n
\r\n
Not Found
```

---

## 📌 まとめ：Step 3 のフロー

1. **`Accept`** で接続を受け入れる
2. **`Read`** でリクエストバイト列を受け取る
3. 先頭行をスペースで分割してメソッドとパスを取り出す
4. パスで分岐してステータスとボディを決める
5. **`Write`** でレスポンスを返す
6. **`Close`** して次の接続へ
