# Step 2：HTTP 最小構成でのレスポンス（前提知識）

Step 1 では TCP で「文字を受け取るだけ」でした。Step 2 では受け取ったデータを無視し、**固定の HTTP レスポンスを返す**ことでブラウザに `Hello, World!` を表示させます。

---

## 1. HTTP はただのテキスト

TCP ソケットが繋がったあとで流れる「文字列のルール」が HTTP です。特別なバイナリ形式ではなく、**プレーンテキストの決まった書式**です。

### ブラウザが送ってくるリクエスト

```
GET / HTTP/1.1\r\n
Host: localhost:8080\r\n
User-Agent: Mozilla/5.0\r\n
\r\n
```

- 1 行目がリクエストライン（メソッド・パス・バージョン）
- 2 行目以降がヘッダー（`フィールド名: 値`）
- **空行（`\r\n` だけの行）でヘッダー終端**

### サーバーが返すレスポンス

```
HTTP/1.1 200 OK\r\n
Content-Length: 13\r\n
\r\n
Hello, World!
```

- 1 行目がステータスライン（バージョン・ステータスコード・テキスト）
- 2 行目以降がレスポンスヘッダー
- 空行のあとにボディ

> ⚠️ 行末は必ず `\r\n`（CR+LF）です。`\n` だけでは RFC 違反で、ブラウザが正しく解釈しないことがあります。

---

## 2. `Content-Length` が重要な理由

ボディが何バイトあるかをブラウザに伝えないと、ブラウザはソケットを読み続けて応答を待ち続けます。`Content-Length` を正確に書くことで「ここまでがレスポンスだ」と伝えます。

Step 2 では固定文字列を返すだけなので、バイト数はあらかじめ計算して定数として書けます。

---

## 3. Step 2 でやること・やらないこと

| やること | やらないこと |
|---|---|
| `conn.Write` でレスポンスを書く | リクエストの内容を解析する |
| `Content-Length` を正しく設定する | パスによるルーティング |
| ブラウザで確認する | ボディの読み取り |

---

## 📌 まとめ：Step 2 のフロー

1. **`Listen`** で受付窓口を開く
2. 無限ループを開始する
3. **`Accept`** で接続を受け入れる
4. **`Read`** でリクエストバイト列を受け取る（内容は使わない）
5. **`Write`** でハードコードしたレスポンス文字列を返す
6. **`Close`** でソケットを閉じてループ先頭に戻る
