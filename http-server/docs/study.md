# ピュアソケットHTTPサーバー自作に必要な知識の整理

ソケット通信によるHTTPサーバーの自作は、大きく分けて**「OS・ネットワークの仕組み」「プロトコルの約束事」「サーバーの構造（並行処理）」**の3つの知識が必要です。

---

## 1. ネットワーク・ソケットの知識（土台）

プログラムがネットワークを介してデータをやり取りするための、OSが提供する仕組み（API）を理解する必要があります。

### 📡 TCP/IP 階層モデルにおける位置づけ
私たちが作るのは、**アプリケーション層（HTTP）**のデータを、OSの**トランスポート層（TCP）**の機能を使って届ける仕組みです。

* **IP（インターネットプロトコル）**: データを目的の「端末（PCやサーバー）」まで届ける役割。
* **TCP（トランスポートプロトコル）**: 届いたデータを「欠損なく、正しい順番で、信頼性高く」やり取りする役割。接続を開き（ハンドシェイク）、データを流し、接続を閉じる。
* **ポート番号**: 端末内の「どのプログラム（プロセス）」にデータを渡すかを識別する番号（例: HTTPは通常 `80`、開発時は `8080` など）。

### 🔌 ソケット（Socket）とは？
OSが用意している**「ネットワークの出入り口（蛇口）」**のようなものです。プログラムからは、ファイルと同じように「開く」「読み込む（Read）」「書き込む（Write）」「閉じる（Close）」という操作ができます。

### 🔄 サーバー側ソケットのライフサイクル
サーバーは以下の手順でソケットを扱い、クライアント（ブラウザ）からの接続を待ち受けます。

1. **Socket（作成）**: ソケットを生成する。
2. **Bind（紐付け）**: ソケットに「IPアドレス」と「ポート番号」を割り当てる。
3. **Listen（待ち受け開始）**: クライアントからの接続要求を受け付ける準備をする（行列の受付を開始するイメージ）。
4. **Accept（接続受け入れ）**: 接続してきたクライアントと1対1で通信するための**「新しいソケット」**を生成して返す。処理はここでクライアントが来るまでブロック（一時停止）します。
5. **Read / Write（データ送受信）**: 繋がったソケットを通じてデータを読み書きする。
6. **Close（切断）**: 通信が終わったらソケットを閉じる。

---

## 2. HTTPプロトコル（仕様・フォーマット）の知識

TCPソケットが繋がったら、その中を流れる「文字列のルール」がHTTPです。HTTPは非常にシンプルで、**「ただのプレーンテキスト（文字列）」**です。



### 📥 HTTPリクエストの構造（ブラウザ ➡️ サーバー）
ブラウザからソケットに送られてくる文字列は、以下のような形をしています。

```http
GET /index.html HTTP/1.1
Host: localhost:8080
User-Agent: Mozilla/5.0
Accept: text/html

（←ここには必ず空行「\r\n」が入る）
